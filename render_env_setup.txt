# Render Dashboard 設定 (alive-checkin service)
# 建立:2026-07-22 04:39
# 用法:打開 Render Dashboard → alive-checkin → Environment → Add Environment Variable

============================================================
目前缺的:LINE_CHANNEL_SECRET (長度 32)
============================================================

把下面這行貼進 Render Environment:

Key  : LINE_CHANNEL_SECRET
Value: <見下方 secrets/render-secret.txt>


============================================================
驗證用 debug endpoint
============================================================

設定完到下面這頁 reload,直到 secret 不再 null:
https://alive-checkin.onrender.com/api/debug/env

============================================================
完整 Render 點擊步驟
============================================================

1. 開瀏覽器 → https://dashboard.render.com
2. 登入 (老闆的 GitHub 帳號)
3. 點左邊 "Services" → 找 "alive-checkin"
4. 點進去 → 左邊 "Environment"
5. 看現在的 Environment Variables 表:
   - 應該有 LINE_CHANNEL_ACCESS_TOKEN (已 OK)
   - 應該沒有 LINE_CHANNEL_SECRET  ← 要補
6. 點右上 "Add Environment Variable"
7. Key 欄填: LINE_CHANNEL_SECRET
8. Value 欄: 從 secrets/render-secret.txt 複製貼上
9. 點 "Save Changes"
10. Render 會自動重啟 service (約 30~60 秒)
11. 重啟完打開: https://alive-checkin.onrender.com/api/debug/env
12. 確認 LINE_CHANNEL_SECRET 三個欄位都不是 null
13. 開 LINE Developers console → webhook 設定 → Verify → 應回 Success

============================================================
注意
============================================================
- Render 不會自動重新部署時還原 secret,只會套用「現在」的值
- 設定後 LINE Developers console 的 webhook URL 應為:
  https://alive-checkin.onrender.com/callback
- secret 補完後,我會再寫一個 commit 把 /api/debug/env endpoint 移除
  (避免以後洩漏資訊)
