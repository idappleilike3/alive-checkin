# 今天還在嗎 MVP 流程說明

## 目前版本定位

這一版先當作 MVP 展示與小規模測試用。目標是先確認使用者願不願意每天簽到、願不願意填緊急聯絡人，以及是否願意為提醒服務付費。

目前資料先用 JSON 檔保存，可以先測市場。等準備正式收費，再升級 PostgreSQL。

## 用戶流程

1. 用戶加入 LINE 官方帳號。
2. 用戶從 LINE LIFF 打開「今天還在嗎」。
3. 第一次進入後，系統建立用戶資料，開始 7 天體驗。
4. 用戶設定每日簽到提醒時間。
5. 用戶至少填 1 位緊急聯絡人。
6. 每天到提醒時間，LINE Bot 推播提醒用戶簽到。
7. 用戶點「今日平安」完成當天簽到。
8. 如果用戶忘記填緊急聯絡人，系統每天早上 9 點會自動提醒。
9. 如果用戶超過寬限時間沒有簽到，後台可以推播逾期提醒。

## 方案流程

目前方案先用人工開通，適合 MVP。

- 7 天體驗：讓用戶先試用。
- 199 活著價：5 位聯絡人 / 好友定位、每日提醒 1 次、LINE 通知、24 小時快照定位。
- 399 安心版：20 位聯絡人 / 好友定位、每日提醒 2 次、LINE 通知、即時追蹤。年付另送 30 天即時追蹤體驗。
- 799 守護版：無限聯絡人 / 好友定位、每日提醒 2 次、LINE + 簡訊 + 電話、軌跡回放、離線同步、專屬客服。

年付採 10 個月價，送 2 個月：

- 199 年付：NT$1,990 / 年
- 399 年付：NT$3,990 / 年
- 799 年付：NT$7,990 / 年

MVP 收費方式：

1. 在 LINE 官方帳號或前台放藍新金流付款連結。
2. 用戶付款後，你到後台手動調整方案。
3. 等市場驗證後，再接藍新付款成功通知，讓系統自動升級方案。

## 後台流程

後台主要給你管理用：

1. 查看總用戶數。
2. 查看今天已簽到人數。
3. 查看逾期用戶。
4. 查看未填緊急聯絡人的用戶。
5. 手動調整用戶方案。
6. 手動推播逾期提醒。
7. 手動提醒用戶補填緊急聯絡人。

平常不用每天手動提醒未填聯絡人，系統已經可以每天早上 9 點自動提醒。

## 自動推播流程

### 未填緊急聯絡人提醒

- 每天台灣時間 09:00 自動檢查。
- 如果用戶緊急聯絡人是 0 位，就推 LINE Bot 提醒。
- 至少填 1 位後，就不會再被這個提醒打擾。

### 每日簽到提醒

- 用戶可以自己設定提醒時間。
- 系統每 15 分鐘檢查一次。
- 到了用戶設定時間，而且今天還沒簽到，才會推播。
- 同一天已經提醒過，就不會一直重複推。

## 好友地圖流程

好友地圖採「雙方同意」模式，不會偷偷定位。

完整定位方案、快照模式、即時追蹤、軌跡回放與年付權益，請看 `定位功能規格.md`。

1. 用戶從 LINE LIFF 開啟服務。
2. 用戶按「產生邀請碼」。
3. 好友輸入邀請碼後，雙方才會成為好友。
4. 用戶按「分享我的位置」後，手機會跳出定位權限。
5. 用戶同意定位後，位置會分享 24 小時。
6. 好友地圖只顯示仍在有效時間內的位置。
7. 用戶可以隨時按「停止分享」。

目前版本不做背景追蹤，只在用戶主動按下分享時更新位置。這樣比較適合 LINE LIFF，也比較符合隱私期待。

## LINE 圖文選單建議

建議做 6 格：

1. 今日簽到
2. 填聯絡人
3. 我的狀態
4. 查看方案
5. 聯絡客服
6. 條款隱私

這樣用戶不用記網址，從 LINE 底部選單就能操作。

## 上線流程

1. 把最新檔案上傳到 GitHub。
2. Render 連接 GitHub repo。
3. Render 建立 Web Service。
4. 設定環境變數：
   - `LIFF_ID`
   - `APP_PUBLIC_URL`
   - `DATA_FILE`
   - `ADMIN_PASSWORD`
   - `LINE_CHANNEL_ACCESS_TOKEN`
   - `APP_TIMEZONE=Asia/Taipei`
   - `CRON_SECRET`
5. Render 建立兩個 Cron Job：
   - 每天 09:00 提醒未填緊急聯絡人。
   - 每 15 分鐘檢查每日簽到提醒。
6. 到 LINE Developers 更新 LIFF endpoint。
7. 用手機從 LINE 打開測試。

## 正式收費前建議補強

1. PostgreSQL：正式保存用戶、簽到、聯絡人與方案資料。
2. 付款自動開通：接藍新金流付款成功通知。
3. 推播紀錄：後台顯示每次推播成功或失敗。
4. 聯絡人同意紀錄：避免未來資料使用爭議。
5. 資料刪除機制：讓用戶可以要求刪除個資。

目前已先補上推播紀錄、資料刪除 API，以及條款/隱私文字補強。PostgreSQL 與藍新金流自動開通建議等公開部署穩定後再接。

## 建議目前先做的順序

1. 先部署 Render，拿到公開網址。
2. 用 LINE LIFF 實機測試。
3. 製作 LINE 圖文選單。
4. 找 5 到 10 位朋友試用。
5. 確認有人願意付費後，再接 PostgreSQL 和藍新自動付款。
## LINE 圖文選單最新版

6 格圖文選單目前採用：

1. 今日簽到
2. 綁定守護人
3. 我的狀態
4. 查看方案
5. 問與答
6. 聯絡客服

第二格已從舊版「填聯絡人」調整為「綁定守護人」。按下後會送出文字「綁定守護人」，由 LINE Bot 回覆一鍵邀請守護人的操作說明。
